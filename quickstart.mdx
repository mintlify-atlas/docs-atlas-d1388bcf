This guide gets the full five-service voting stack running on your machine. You will cast a vote, watch it flow through Redis and the .NET worker, and see live results update in your browser.

## Prerequisites

<Tabs>
  <Tab title="Mac / Windows">
    Install [Docker Desktop](https://www.docker.com/products/docker-desktop). Docker Compose is bundled and enabled by default — no extra steps required.

    Verify your installation:

    ```bash
    docker --version
    docker compose version
    ```
  </Tab>
  <Tab title="Linux">
    Install Docker Engine and the Compose plugin separately.

    ```bash
    # Install Docker Engine (Ubuntu/Debian)
    sudo apt-get update
    sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    ```

    Add your user to the `docker` group so you can run commands without `sudo`:

    ```bash
    sudo usermod -aG docker $USER
    newgrp docker
    ```

    Verify your installation:

    ```bash
    docker --version
    docker compose version
    ```

    <Info>
      The Compose plugin ships as `docker compose` (with a space). The legacy standalone `docker-compose` binary is not required.
    </Info>
  </Tab>
</Tabs>

## Run the app

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/dockersamples/example-voting-app.git
    cd example-voting-app
    ```
  </Step>

  <Step title="Start all services">
    Build the images and start every container in the foreground:

    ```bash
    docker compose up
    ```

    Docker Compose will build three images (`vote`, `result`, `worker`) and pull two official images (`redis:alpine`, `postgres:15-alpine`). The first run takes a few minutes; subsequent starts are fast because the layers are cached.

    You will see interleaved log output from all five services. The stack is ready when you see lines similar to:

    ```
    vote-1    |  * Running on http://0.0.0.0:80
    result-1  | App running on port 4000
    worker-1  | Connected to db
    worker-1  | Connecting to redis
    ```

    <Info>
      The `worker` service retries its Redis and Postgres connections until they pass their health checks. Brief `Waiting for db` or `Waiting for redis` messages during startup are normal.
    </Info>
  </Step>

  <Step title="Open the voting and results pages">
    With the stack running, open two browser tabs:

    | Service | URL | Description |
    |---------|-----|-------------|
    | Vote | [http://localhost:8080](http://localhost:8080) | Cast a vote for Cats or Dogs |
    | Results | [http://localhost:8081](http://localhost:8081) | Watch live vote totals update |

    Click **Cats** or **Dogs** on the vote page. Switch to the results tab — the count updates within about one second as the .NET worker processes the message from Redis and writes it to Postgres.
  </Step>

  <Step title="(Optional) Seed the database with test votes">
    To pre-populate the database with a batch of votes, start the `seed` service using the `seed` profile. This service is excluded from the default `up` command and runs once then exits.

    ```bash
    docker compose --profile seed up -d
    ```

    The seed container submits votes against the `vote` service's HTTP endpoint and terminates automatically after it finishes. Check the results page to see the seeded totals.
  </Step>

  <Step title="Stop the app">
    Press `Ctrl+C` in the terminal where `docker compose up` is running, then remove the containers and network:

    ```bash
    docker compose down
    ```

    This stops and removes the containers but preserves the `db-data` named volume so your votes survive a restart. To remove the volume as well:

    ```bash
    docker compose down --volumes
    ```
  </Step>
</Steps>

## How voting works

<Note>
  The vote app tracks voters by a `voter_id` cookie set in the browser. Each browser (or browser profile) counts as one voter — submitting multiple votes from the same browser only updates your existing vote rather than adding a new one. To simulate multiple distinct voters, open the vote page in different browsers or use private/incognito windows.
</Note>

When you click a vote button, the Python Flask app (`vote/app.py`) pushes a JSON message to the `votes` Redis list:

```json
{"voter_id": "<hex-id-from-cookie>", "vote": "a"}
```

The .NET worker (`worker/Program.cs`) polls the `votes` list every 100 ms, dequeues each message, and upserts a row into the `votes` table in Postgres:

```sql
INSERT INTO votes (id, vote) VALUES (@id, @vote)
-- or, if the voter_id already exists:
UPDATE votes SET vote = @vote WHERE id = @id
```

The Node.js result server (`result/server.js`) queries Postgres once per second and broadcasts the totals to every connected browser over Socket.IO, so the results page updates in real time without polling.
